## Goal

One focused pass over the open-source vibekit config repo that: (1) encodes a
no-AI-attribution authorship rule, (2) removes accumulated rules drift, (3)
documents the repo's real security/trust model, (4) applies a few small tested
code hardening fixes, and (5) scrubs personal infra references from the public
config â€” all delivered through the existing `brainstorm â†’ spec â†’ ship` loop.

vibekit is a **public** repo. Every change is evaluated against that: no private
infra, no secrets, no personal identifiers added; sharp defaults documented.

## Decisions (locked)

- Caveman installer: **pin to a commit SHA** (overridable via `CAVEMAN_INSTALL_URL`).
- `Bash(*)` + `acceptEdits`: **keep** (deliberate personal default), document the tradeoff.
- memex memory block in public `claude/CLAUDE.md`: **genericize** (drop personal backend/repo).
- Gap rules: **authorship only** this pass; other gaps listed as opt-in backlog.

## Deliverables

### A. Authorship rule (primary ask)

New files:
- `claude/rules/authorship.md`
- `cursor/rules/authorship.mdc` (with the `.mdc` frontmatter: `description`, `globs`, `alwaysApply: true`, matching sibling rules)

One-line pointer added to `CLAUDE.md` and `claude/CLAUDE.md` referencing the rule.

Rule content (intent):
- Code comments are authored in the human owner's voice. Sparse; explain *why*,
  not *what* (consistent with the existing Documentation section). No
  AI/assistant self-reference, no "generated by", no chatbot-style narration.
- Commits: **no** `Co-Authored-By: Claude` / AI co-author trailer.
- PRs: **no** "Generated with Claude Code" footer or AI attribution.
- This rule intentionally overrides any harness/tool default that injects such
  attribution (user instructions outrank tool defaults).

Kiro: `kiro/agents/default.json` references the same rule set; confirm it points
at the rules directory so the policy is inherited (no separate copy if it reads
from a shared location; otherwise add a short equivalent note).

### B. Rules hygiene

One-time cleanup of **live** `~/.claude/rules/` orphans not tracked in vibekit
(confirmed stale/duplicate by analysis):
- `coding-style.md`  â†’ duplicate of `style.md`

No auto-prune logic is added to `install.sh` (deleting files during sync is
risky; YAGNI). The orphan deletion is a documented one-time manual step.

### D. Targeted code fixes (tested)

1. `claude/hooks/vibenotif.py` + `cursor/hooks/vibenotif.py`: before POSTing,
   validate each configured target URL has an `http`/`https` scheme and skip
   (with a warning) anything else. Only send the bearer token to the configured
   trusted origin. Small, behavior-preserving for valid configs.
2. `lib/vibemon.sh`: guard the `rm -rf` paths so they only run when `$HOME` is
   non-empty and the target is under the expected `$HOME/.npm/_npx/` prefix.

Tests added under `test/` following the existing `test/run.sh` + `helpers.sh`
pattern (shell tests for vibemon guard; a focused Python check for URL
validation, or a shell harness around the hook if simpler).

### E. Privacy scrub

keeps the personal detail; only the tracked public copy is genericized. This
follows the user's own two-gate OSS-publication scrub standard.
